# Інформаційна система «Бюро знайомств»

Це курсовий проєкт з розробки програмного забезпечення для інформаційної системи «Бюро знайомств».

Система призначена для автоматизації основних процесів роботи бюро знайомств: обліку клієнтів і працівників, створення пар, розрахунку сумісності, організації зустрічей, ведення архіву пар і пошуку потенційних партнерів за заданими критеріями.

## Структура проєкту

Проєкт складається з двох основних частин:

* `DatingAgency` — клієнтська частина на Windows Forms;
* `DatingAgencyServer` — серверна частина на ASP.NET Core Web API.

Клієнтська частина відповідає за графічний інтерфейс користувача. Серверна частина приймає HTTP-запити, виконує бізнес-логіку, обробляє дані та взаємодіє з базою даних SQL Server.

Додатково в репозиторії є папка:

* `Database` — SQL-скрипт для створення бази даних і початкового заповнення.

## Ролі користувачів

У системі передбачено три ролі користувачів:

* клієнт;
* працівник;
* адміністратор.

Клієнт може переглядати анкети потенційних партнерів і виконувати пошук за віком, зростом, містом та освітою.

Працівник може переглядати клієнтів, створювати пари, додавати клієнтів і організовувати зустрічі.

Адміністратор може переглядати основні дані системи, керувати працівниками, створювати, видаляти й архівувати пари, а також змінювати статус активності працівників.

## Архітектура

Система побудована за клієнт-серверною багаторівневою архітектурою з елементами мікросервісного підходу.

Клієнтська частина не звертається напряму до бази даних. Уся взаємодія з даними виконується через серверну частину.

Загальна схема роботи системи:

```text
Windows Forms → ApiClient → HTTP-запит → Controller → Service → Repository → SQL Server
```

Серверна частина `DatingAgencyServer` запускається окремо, має власні API-маршрути та може бути перевірена через Swagger.

## Налаштування бази даних

Перед запуском проєкту потрібно створити базу даних `DatingAgency` у SQL Server.

SQL-скрипт для створення бази даних і початкових даних розміщено в папці:

```text
Database/DB_2026_DatingAgency.sql
```

Для створення бази даних потрібно відкрити цей файл у MS SQL Server Management Studio та виконати скрипт.

Також потрібно перевірити рядок підключення до бази даних у файлі:

```text
DatingAgencyServer/appsettings.json
```

Приклад рядка підключення:

```json
"ConnectionStrings": {
  "DefaultConnection": "Data Source=localhost\\SQLEXPRESS;Initial Catalog=DatingAgency;Integrated Security=True;TrustServerCertificate=True"
}
```

Якщо SQL Server має іншу назву, потрібно змінити частину:

```text
localhost\SQLEXPRESS
```

на назву власного SQL Server.

## Запуск проєкту

Для запуску системи потрібно:

1. Відкрити файл `DatingAgency.sln` у Visual Studio 2022.
2. Переконатися, що база даних `DatingAgency` створена в SQL Server.
3. Перевірити рядок підключення у файлі `DatingAgencyServer/appsettings.json`.
4. Запустити серверну частину `DatingAgencyServer`.
5. Переконатися, що відкрився Swagger і сервер працює.
6. Запустити клієнтську частину `DatingAgency`.
7. Увійти до системи під відповідною роллю користувача.
